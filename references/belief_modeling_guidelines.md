# Belief Input Guidelines For Dialogue Tool

This document defines how beliefs should be provided to `generate_event_dialogues`.

## Current Role Of Beliefs

In the current FastMCP backend:

- beliefs are input context for intention refinement and dialogue simulation
- beliefs are not generated by a separate MCP tool yet

## Input Format

`beliefs` should be a JSON list. Recommended item schema:

```json
{
  "belief_id": "belief_001",
  "triple": ["user", "prefers", "quiet_commute"],
  "description": "User recently prefers less crowded commuting routes.",
  "belief_type": "situational",
  "time": "2025-02-12",
  "confidence": 0.84,
  "salience": 0.79
}
```

Minimal valid input can also be:

- list of strings, or
- list of lightweight dicts

The dialogue tool treats beliefs as contextual evidence text.

## Practical Recommendations

- Keep beliefs temporally aligned with input events.
- Prioritize high-salience beliefs to reduce prompt noise.
- Avoid contradictory beliefs in same request.
- Update beliefs between calls if running multi-stage simulation.

## Suggested Next Extension

If you later add a third tool, prefer:

- `generate_beliefs_from_events_and_dialogues(user_profile, events, dialogues, previous_beliefs)`

Then chain:

1. life events
2. dialogues
3. belief update

for iterative long-horizon simulation.
